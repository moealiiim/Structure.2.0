Personalhanteringssystem

Kunskapsdomän: Kompetensbaserad personalorganisation  |  MySQL + Node.js  |  2026

Huvud-entiteter (5 st)
EMPLOYEES
PK employee_id
FK department_id
first_name, last_name
UQ email
salary (DECIMAL) 💰
hire_date (DATE) 📅
employment_type (ENUM)
DEPARTMENTS
PK department_id
UQ name
budget (DECIMAL) 💰
ROLES
PK role_id
title, seniority_level
TASKS
PK task_id
title, priority (ENUM)
SKILLS
PK skill_id
name, category
Kopplingstabeller (M:N)
EMPLOYEE_ROLES
FK employee_id
FK role_id
start_date (DATE) 📅
end_date (DATE) 📅
ROLE_TASKS
FK role_id
FK task_id
ROLE_SKILLS
FK role_id
FK skill_id
required_level (ENUM)
💡 Composite PK på alla kopplingstabeller förhindrar dubbletter automatiskt
Relationer & Kardinalitet
employees → departments N:1
En anställd tillhör EN avdelning
employees ↔ roles M:N
Via employee_roles
roles ↔ tasks M:N
Via role_tasks
roles ↔ skills M:N
Via role_skills
Tekniska val
Databas: MySQL 8.0
Relationell data med tydliga FK-kopplingar. Stöd för transactions, prepared statements och 3NF.
Backend: Node.js + mysql2
Async/await, prepared statements med execute(), dotenv för miljövariabler.
Säkerhet:
✓ .env för lösenord
✓ .gitignore skyddar .env
✓ Prepared statements
✓ Input-validering i models
Kunskapsdomän
Domän:
Kompetensbaserad personalorganisation
Problemet:
Företaget vet inte vem som gör vad, vilka kompetenser som finns eller hur man hittar rätt person för rätt uppgift.
Lösningen:
Relationsdatabas med 5 entiteter och 3 kopplingstabeller som kopplar samman personal, roller, uppgifter och kompetenser.
